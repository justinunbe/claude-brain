# Session 6 — Skills for the Memory System

Second session of Phase 1 on the Personal AI Partner project. The question on the table: now that the memory structure exists, how do we actually keep it in use? Jay framed it as a skills problem — which Claude skills are needed so the documentation stays current and gets referenced correctly across every kind of situation.

## The question Jay asked

Two-part: (1) What are all the situations in which the brain should be updated or added to? (2) When should those updates take place? Both to be solved through skills.

## First-pass map

Organized the update situations by cadence: continuous (silent capture within a session), session-boundary (bootstrap at start, synthesis at end), cadenced (weekly planning, monthly consolidation), async (Jay's backchannel, direct edits), and external (future ingestion). Proposed 4-5 skills covering the space.

## Two rules Jay drew

Both rejected assumptions in the first-pass map and simplified the whole model:

1. **Jay will not push to the repo independently.** Every commit and push flows through Claude. If that ever changes, Jay says he'll notify.
2. **Jay will not write directly in the .md files.** The brain is Claude's to write. That applied to `profile/from-jay.md` and any other file I was going to have Jay edit.

Effect: the whole "async / external divergence" lane collapsed. No backchannel file. No bootstrap logic for "did Jay edit something directly." No "repo diverged from upstream" check. Every update is session-driven, confirmed by Jay, written by Claude, committed through a command Jay runs in Terminal. One mental model, no branches.

## Architecture decided

Four skills cover the system:

- **`brain-bootstrap`** — runs at session start. Reads profile, state, current hare's area file, latest session. Replaces `notion-brain`.
- **`brain-pull`** — mid-session topic-specific retrieval. When conversation pivots to an Area or topic not in bootstrap's read set, load the relevant file(s) before responding.
- **`session-synthesis`** — runs at session end. Surfaces the proposed diff, Jay confirms, Claude writes, outputs the exact Terminal commit+push command.
- **`consolidate-memory`** (retargeted from Notion to the repo) — monthly reflective pass. Walks `confirmed` dates, surfaces duplicates and contradictions, proposes pruning list.
